## 1. 技术栈选型

### 1.1 核心框架与库

| 技术选型 | 版本 | 选型理由 |
|---------|------|---------|
| **React** | 18.x | - 组件化开发，复用性高<br>- 生态系统成熟，社区活跃<br>- 虚拟DOM，性能优秀<br>- 企业级应用首选框架 |
| **TypeScript** | 5.x | - 类型安全，减少运行时错误<br>- 更好的开发体验和代码提示<br>- 提升代码可维护性和可扩展性<br>- 大型团队协作首选 |
| **Next.js** | 14.x | - 基于React的全栈框架<br>- 支持SSR/SSG，SEO友好<br>- 文件系统路由，开发效率高<br>- 内置API路由，便于与后端集成 |
| **Tailwind CSS** | 3.x | - 原子化CSS，开发效率极高<br>- 响应式设计支持完善<br>- 自定义主题配置灵活<br>- 生成的CSS体积小，性能好 |

### 1.2 UI组件库与工具

| 技术选型 | 版本 | 选型理由 |
|---------|------|---------|
| **shadcn/ui** | 最新 | - 基于Radix UI，可访问性优秀<br>- 组件设计精美，现代化<br>- 完全可定制，无样式锁定<br>- 与Tailwind CSS完美配合 |
| **React Hook Form** | 7.x | - 高性能表单管理<br>-  uncontrolled组件模式<br>- 极小的包体积<br>- 完美的TypeScript支持 |
| **Zod** | 3.x | - TypeScript优先的 schema 验证<br>- 类型安全的验证<br>- 与React Hook Form完美集成<br>- 极小的运行时开销 |
| **Framer Motion** | 11.x | - 声明式动画库<br>- 强大的动画能力<br>- 优秀的性能<br>- React友好的API |

### 1.3 状态管理与数据流

| 技术选型 | 版本 | 选型理由 |
|---------|------|---------|
| **Zustand** | 4.x | - 极简的状态管理<br>- 无样板代码<br>- 优秀的TypeScript支持<br>- 高性能，无不必要的重渲染 |
| **React Query (TanStack Query)** | 5.x | - 服务端状态管理<br>- 自动缓存和重新获取<br>- 乐观更新支持<br>- 完美的分页和无限滚动支持 |

### 1.4 图表与数据可视化

| 技术选型 | 版本 | 选型理由 |

| **D3.js** | 7.x | - 强大的数据可视化能力<br>- 用于复杂的自定义图表<br>- 与React良好集成 |

### 1.5 甘特图与项目管理

### 4.2 核心交互流程

#### 4.2.1 新建项目流程

**交互设计：**

1. **触发点**：点击首页"新建项目"按钮
2. **步骤1 - 选择项目类型**
   - 模态框展示项目类型卡片
   - 支持搜索和快速筛选
   - 每个类型显示模板预览
3. **步骤2 - 填写基本信息**
   - 向导式表单，分步填写
   - 表单验证实时提示
   - 支持保存草稿
4. **步骤3 - 配置团队**
   - 团队成员选择器
   - 角色分配
   - 权限预览
5. **步骤4 - 确认创建**
   - 项目信息概览
   - 模板配置预览
   - 创建按钮

**视觉反馈：**
- 步骤指示器
- 加载动画
- 成功提示
- 项目创建动画

#### 4.2.2 进度更新流程

**交互设计：**

1. **甘特图交互**
   - 拖拽任务条调整时间
   - 拖拽调整依赖关系
   - 缩放时间轴
2. **任务更新**
   - 点击任务弹出侧边栏
   - 进度滑块
   - 状态选择器
   - 备注输入
3. **批量操作**
   - 多选任务
   - 批量更新状态
   - 批量分配负责人

**视觉反馈：**
- 拖拽时的视觉引导
- 更新后的动画效果
- 冲突警告提示
- 保存状态指示器

#### 4.2.3 审批流程

**交互设计：**

1. **审批发起**
   - 一键发起审批
   - 审批人选择
   - 消息通知
2. **审批处理**
   - 待审批列表
   - 审批详情面板
   - 审批操作（通过/驳回）
   - 意见填写
3. **审批跟踪**
   - 审批进度条
   - 审批历史
   - 状态通知

**视觉反馈：**
- 审批状态标签
- 实时通知徽章
- 审批流程动画
- 完成庆祝效果

### 4.3 响应式设计

#### 4.3.1 断点设计

| 断点 | 设备类型 | 布局调整 |
|------|---------|---------|
| < 640px | 手机 | 单列布局，侧边栏折叠为汉堡菜单 |
| 640-1024px | 平板 | 两列布局，侧边栏可折叠 |
| > 1024px | 桌面 | 三列布局，完整侧边栏 |

#### 4.3.2 移动端适配策略

- **触摸友好**：按钮最小44x44px
- **手势支持**：滑动返回、下拉刷新
- **性能优化**：图片懒加载、组件懒加载
- **离线支持**：PWA支持

#### 4.4.1 关键可访问性特性

```typescript
// src/services/api.ts
import axios from 'axios'

const api = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_URL,
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json',
  },
})

// 请求拦截器
api.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem('token')
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    return config
  },
  (error) => Promise.reject(error)
)

// 响应拦截器
api.interceptors.response.use(
  (response) => response.data,
  (error) => {
    if (error.response?.status === 401) {
      // 处理未授权
    }
    return Promise.reject(error)
  }
)

export default api
```
