## 1. 文档概述

### 1.1 文档目的
本文档详细描述"项目管理一体化平台"的产品需求，包括功能规格、业务流程、界面原型、非功能性需求等，作为开发、测试、设计团队的执行依据。

### 1.2 项目背景
根据《贵州省国资委监管企业数字化监管项目代码工作指引（试行）》要求，公司立项的项目需要赋项目代码。当前项目管理涉及投资计划、立项、采购、合同、进度、成本、验收等多个环节，业务分散在OA、财务、采购、档案等多个独立系统中，存在数据分散、流程割裂、管控滞后等痛点。

### 1.3 目标用户

| 用户角色 | 核心诉求 | 使用频率 |
|---------|---------|---------|
| 公司领导 | 全局视图、决策支持、风险预警 | 每日查看驾驶舱 |
| 部门负责人 | 部门项目进度、资源协调、审批决策 | 每日处理待办 |
| 项目经理 | 项目全生命周期管理、团队协作 | 高频使用 |
| 项目成员 | 任务执行、文档协作、进度汇报 | 每日使用 |
| 采购专员 | 采购计划、执行、供应商管理 | 按需使用 |
| 合同管理员 | 合同起草、审批、履约跟踪 | 高频使用 |
| 财务人员 | 预算控制、付款审核、成本分析 | 定期使用 |
| 风控合规人员 | 合规审查、风险监控、审计追溯 | 按需使用 |

## 2. 产品定位与目标

### 2.1 产品定位
构建一个**覆盖项目全生命周期、全要素管理、全组织贯通的穿透式项目管理一体化平台**，实现：
- 业务在线化
- 数据资产化
- 决策智能化
- 管理穿透化

### 2.2 核心价值主张

```
┌─────────────────────────────────────────────────────────────┐
│                      核心价值主张                           │
├─────────────────────────────────────────────────────────────┤
│  🎯 一码贯通    项目代码贯穿全生命周期，数据不落地            │
│  🔗 业财一体    项目-预算-合同-付款全流程打通                │
│  🤖 AI赋能      智能预警、文档识别、辅助决策                 │
│  📊 穿透管理    上级看下级，数据层级穿透，决策有依据         │
│  ⚡ 合规内控    三重一大、审计监管、内控要求全满足           │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 成功指标

| 指标类型 | 具体指标 | 目标值 |
|---------|---------|--------|
| 效率指标 | 项目审批周期 | 缩短 50% |
| 效率指标 | 报表编制时间 | 减少 70% |
| 质量指标 | 流程合规率 | ≥ 98% |
| 质量指标 | 合同关键条款缺失率 | 降至 0 |
| 满意度 | 用户满意度 | ≥ 4.5/5 |

### 4.4 立项管理

#### 4.4.1 功能描述
提供不同项目类型的标准化申请表单，支持附件上传，作为项目创建起点。

#### 4.4.2 项目类型与表单

| 项目类型 | 适用场景 | 特殊字段 |
|---------|---------|---------|
| 投资类项目 | 固定资产投资、股权投资 | 投资回收期、预期收益率 |
| 设备类项目 | 设备采购、设备改造 | 设备清单、技术参数 |
| 科研类项目 | 技术研发、产品创新 | 知识产权归属、成果形式 |
| 信息化类项目 | IT系统建设、数字化转型 | 系统架构、数据安全等级 |
| 运维类项目 | 设备维护、系统运维 | 服务周期、SLA要求 |

#### 4.4.3 AI能力
- **字段自动填充**：根据项目名称和类型，自动推荐项目类型、预算科目
- **材料完整性检查**：识别上传附件（可研报告），检查关键章节是否齐全并提示

### 4.5 采购管理

#### 4.5.1 功能描述
覆盖采购计划、采购申请、采购执行的全流程管理。

#### 4.5.2 采购流程

```
采购计划编制 → 采购申请 → 采购文件编写 → 采购文件审核 → 采购执行 → 结果录入
```

#### 4.5.3 AI能力
- **供应商智能推荐**：从供应商库中，基于采购品类、历史合作评价、履约能力智能推荐供应商
- **智能审核助手**：基于知识库审核要点进行审核，分为提醒、警告类型
- **投标智能识别**：识别投标文件客观评分部分资料，智能审核并汇总评分结果

### 4.6 合同管理（核心模块）

#### 4.6.1 功能描述
覆盖合同全生命周期：相对方管理 → 合同起草 → 审批 → 履约 → 变更 → 归档。

#### 4.6.2 核心功能

| 功能 | 描述 |
|------|------|
| 相对方库 | 相对方台账，支持企查查等API自动查询填充信息 |
| 合同模板库 | 法务部门维护标准合同范本，支持版本控制 |
| 合同起草 | AI辅助起草，自动填充已知信息，提示必备条款 |
| 智能审查 | 条款完整性检查、风险点标定、相对方征信速览 |
| 履约管理 | 自动提取履约节点，形成履约日历，逾期预警 |
| 电子签章 | 符合《电子签名法》的标准化电子印章 |

#### 4.6.3 智能审查规则

| 检查项 | 检查内容 | 预警级别 |
|--------|---------|---------|
| 必备条款 | 保密条款、知识产权条款、争议解决条款 | 警告 |
| 风险条款 | 过高违约金、无限责任、单方解约权 | 严重 |
| 相对方风险 | 诉讼记录、失信信息、经营异常 | 提醒 |
| 范本偏离 | 使用非范本或重大修改 | 警告 |

### 4.7 智能预警

#### 4.7.1 预警规则库

| 预警类型 | 触发条件 | 推送方式 | 接收人 |
|---------|---------|---------|--------|
| 合同到期 | 到期前30天、10天 | 企业微信+站内信 | 经办人、负责人 |
| 进度滞后 | 实际进度 < 计划进度-10% | 企业微信+站内信 | 项目经理、部门领导 |
| 预算超支 | 成本偏差率 > 10% | 企业微信+邮件 | 项目经理、财务人员 |
| 付款逾期 | 应付未付超约定时间 | 企业微信+站内信 | 经办人、财务人员 |
| 资质到期 | 相对方资质到期前30天 | 站内信 | 合同管理员 |

## 5. 非功能性需求

### 5.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面响应时间 | 核心页面 < 2秒，复杂查询 < 5秒 |
| 并发用户数 | 支持 500 用户同时在线 |
| 系统吞吐量 | 支持日活 2000 用户 |
| 数据导出 | 支持单次导出 10万 条记录 |

### 5.2 安全需求

| 要求 | 说明 |
|------|------|
| 等保认证 | 通过国家网络安全等级保护三级认证 |
| 数据加密 | 传输加密(TLS 1.3)、存储加密(AES-256) |
| 权限控制 | RBAC模型，支持数据权限、菜单权限、操作权限 |
| 审计日志 | 所有数据访问记录日志，支持追溯 |
| 敏感数据 | 项目敏感级别设置（公开/内部/秘密） |

### 5.3 可用性需求

| 指标 | 要求 |
|------|------|
| 系统可用性 | ≥ 99.5% |
| 故障恢复 | RTO < 4小时，RPO < 1小时 |
| 备份策略 | 每日增量备份，每周全量备份 |

### 5.4 可扩展性需求

- 采用微服务架构，支持业务功能灵活扩展
- 支持水平扩展，应对业务增长
- 预留SaaS化演进接口
